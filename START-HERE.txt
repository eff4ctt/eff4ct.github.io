🚀 DÉMARRAGE ULTRA-RAPIDE

Étape 1 : Sur GitHub, créez un repository public
        → Nom : votre-username.github.io (ou autre)

Étape 2 : Dans votre terminal :
        cd cybersec-blog
        git init
        git add .
        git commit -m "Initial commit"
        git branch -M main
        git remote add origin https://github.com/USERNAME/REPO.git
        git push -u origin main

Étape 3 : Sur GitHub → Settings → Pages → Source: GitHub Actions
        ✅ Le site se déploie automatiquement !

Étape 4 : Configuration du nom de domaine (voir NAMECHEAP-GUIDE.md)

✏️  PERSONNALISATION AVANT LE PREMIER DÉPLOIEMENT

1. Éditez hugo.toml :
   → baseURL = 'https://votre-domaine.com/'
   → title = 'Votre Nom - Cybersecurity Blog'
   → author = 'Votre Nom'

2. Éditez static/CNAME :
   → Mettez votre nom de domaine

3. Éditez content/about/_index.md :
   → Ajoutez vos infos, contact, certifications

4. (Optionnel) Supprimez les articles exemples dans content/posts/

📝 CRÉER UN NOUVEL ARTICLE

1. Créez un fichier dans content/posts/
   → Exemple : mon-writeup.md

2. Structure d'un article :

   ---
   title: "Titre de votre writeup"
   date: 2025-10-29
   draft: false
   ---

   ## Introduction
   Votre contenu...

   ## Exploitation
   ```python
   # Votre code
   ```

   ## Conclusion

3. Déployez :
   git add content/posts/mon-writeup.md
   git commit -m "Add new writeup"
   git push

🌐 CONFIGURATION NAMECHEAP (RÉSUMÉ)

Option A - Domaine principal (example.com) :
   Advanced DNS → Ajoutez 5 enregistrements :
   • 4 × A Record → @ → 185.199.108-111.153
   • 1 × CNAME → www → votre-username.github.io

Option B - Sous-domaine (blog.example.com) :
   Advanced DNS → Ajoutez 1 enregistrement :
   • 1 × CNAME → blog → votre-username.github.io

Sur GitHub :
   Settings → Pages → Custom domain → votre-domaine.com
   (Attendez 10 min puis cochez "Enforce HTTPS")

Problème : Le site ne se déploie pas
